# Відповіді на питання для самоперевірки

## 1. Чим цей чат принципово відрізняється від чату на HTTP-запитах?

WebSocket створює постійне двостороннє з'єднання між клієнтом і сервером. Сервер може самостійно надсилати повідомлення клієнтам у будь-який момент.

HTTP працює за схемою «запит-відповідь», тому клієнт повинен постійно надсилати нові запити для отримання оновлень. Через це HTTP менш ефективний для чатів у реальному часі.

---

## 2. Що означає схема ws:// і чим відрізняється wss://?

- `ws://` — звичайне WebSocket-з'єднання без шифрування.
- `wss://` — захищене WebSocket-з'єднання з використанням TLS/SSL, аналог HTTPS.

---

## 3. Що треба змінити, щоб надсилати повідомлення лише одному конкретному користувачу?

Потрібно зберігати інформацію про підключених клієнтів та їхні унікальні ідентифікатори. Потім повідомлення надсилати не всім через `wss.clients.forEach()`, а лише потрібному клієнту за його ID.

---

## 4. Чому в завданні 5 час додається на сервері, а не на клієнті?

Сервер є єдиним джерелом часу для всіх користувачів. Якщо додавати час на клієнті, то через різні часові пояси або неправильні налаштування системного часу повідомлення можуть мати різні часові позначки. Додавання часу на сервері забезпечує однаковий та коректний час для всіх клієнтів.
